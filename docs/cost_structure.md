# 成本结构说明

## 核心原则

本项目的成本结构严格隔离为两条管线：**业务运行管线（¥0）** 与 **学术包装管线（¥70）**。

## 业务管线：¥0 / 月

QClaw 分析系统的日常运营成本为零，依赖 DeepSeek 平台的每日 800 积分免费额度调度：

| 环节 | 单次消耗 | 模型 |
|:--|:--|:--|
| D（拆解） | ~15 积分 | DeepSeek V4 Flash |
| E（评估） | ~35 积分 | DeepSeek V4 Flash |
| A（聚合） | ~20 积分 | DeepSeek V4 Flash |
| 批量诊断（每批次） | ~200 积分 | DeepSeek V4 Pro |
| **每 PV 全流程** | **~70 积分** | — |

按每日 800 积分额度、单 PV ~70 积分计算，**日均可处理 10+ PV**，远超运营需求（游戏版本更新频率为周级）。

## 路由降本逻辑

```
任务输入 → [Flash 执行] D→E→A → 产出
                ↓ 批次完成后
           [V4 Pro 批量诊断] → 方法论审查 → 长期记忆注入
```

- **执行层始终用 Flash**（快、便宜、免费额度充足）
- **诊断层仅在批次完成后触发 Pro**（不实时诊断，节省 ~60% 诊断开销）
- **自反思缓冲层已移除**（实验证实净效应为零，纯浪费积分）

## 学术管线：¥70（一次性）

CC 专项（ChinaXiv）的预印本审核费，与业务管线物理隔离。该费用为学术出版基础设施开销，不影响业务系统的零成本属性。

## 工业外推边界

本项目的 ¥0 成本建立在 DeepSeek 平台的免费额度之上。在工业环境中外推时：

- **自建模型部署**：需计入 GPU 推理成本，但 QClaw 的 800 积分/日调度逻辑等价于"轻量模型执行 + 强模型按需诊断"的路由策略，可直接映射到混合推理架构
- **第三方 API**：按 Token 计费时，Flash/Pro 的分层调用策略直接转化为成本控制机制——执行层用低价模型，诊断层用高价模型但仅在批次边界触发
- **成本上限**：路由配置中的 `max_daily_tokens` 和 `diagnosis_batch_threshold` 参数提供了硬性成本天花板

## 不适用场景

- 实时性要求极高的在线服务（本架构假设批次处理，延迟容忍度为小时级）
- 单任务独立诊断场景（批量诊断的经济性来自跨实例比较，单任务无此优势）
