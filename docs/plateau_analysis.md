# 自动化收益拐点分析

## 现象

QClaw 系统在 V2.0 后出现指标走平：PASS 率稳定在 93-97%，E/(D+A) 比率稳定在 2.5-3.0 区间，诊断轮次的方法论改进边际递减。

## 这不是方法失效

V2.0 后的指标走平是 **协议收敛边界**（protocol convergence boundary），而非 EDOT 方法失效。原因：

1. **D 瓶颈已基本消除**：V1→V2 prompt 改进将 D 类缺陷压缩 -91%（new36），可继续压缩的空间有限
2. **E 瓶颈存在不可压缩下限**：跨 5+ 独立数据集，E 类缺陷始终维持在 27~34 范围——这不是 prompt 问题，是评估环节固有的 3 次类型转换（定性→定量、绝对→相对、描述→指令）带来的结构性成本
3. **诊断模型的相对优势收窄**：当执行 prompt 已经吸收了前几轮诊断的改进后，诊断模型能发现的新问题自然减少

## 拐点判断标准

| 指标 | 阈值 | 含义 |
|:--|:--|:--|
| PASS 率变化 | 连续 3 轮 <2pp 变动 | 执行质量接近当前模型能力上限 |
| E/(D+A) 变化 | 连续 3 轮在 ±0.3 内波动 | E 瓶颈已从"可修"进入"结构性"区间 |
| 诊断新增缺陷数 | 连续 2 轮 <5 条 | 方法论改进空间耗尽 |

## 自动/人工切换 SOP

```
连续 2 轮满足拐点条件
       ↓
  [自动] 降频诊断：从"每批诊断"切换为"每 3 批诊断"
       ↓
  [自动] 维持长期记忆注入（当前方法论已固化）
       ↓
  连续 5 轮指标无退化 → 系统进入稳态维护模式
       ↓
  [人工介入触发条件]：
    - PASS 率单轮骤降 >10pp  → 外部环境变化（新任务类型/数据分布偏移）
    - E/(D+A) 单轮跃升 >1.0  → 评估标准可能需要升级
    - 用户反馈负面趋势        → 业务需求变化
```

## 边界声明

拐点分析基于 QClaw 系统的 22 轮诊断追踪数据，样本量有限（n=22 轮，单系统）。不同领域、不同模型组合的拐点位置可能不同。本文档提供的阈值和 SOP 是参考框架，非通用标准。
